Track per-generation evolution metrics for convergence analysis (K=1 vs K=3 etc.).

Record a small set of metrics each generation and expose their history so we can read convergence *curves* instead of eyeballing top-10 snapshots:
- max fitness (best individual)
- average fitness (population mean)
- diversity

Diversity metric: start with lexical diversity = unique words / total words across the whole population's genomes (ratio in (0,1]; ~1 = all distinct, low = converged/repetitive). Cheap to compute from genome unit lists. Consider alternatives later (count of distinct genomes, mean pairwise word-overlap / Jaccard distance between genomes) if word-ratio is too coarse.

Implementation sketch:
- evolution.py already keeps fitness_history (best per gen). Add parallel histories (e.g. avg_fitness_history, diversity_history) appended in evolve_generation alongside the existing best-fitness append, and seed them in initialize_population.

- Optional follow-up: chart the curves in the UI.

Motivation: comparing tournament_size=1 vs 3 needs the diversity-decay and best-fitness-climb trajectories on the SAME target across seeds, not single-run snapshots.
